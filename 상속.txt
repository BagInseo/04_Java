상속과 다형성의 목적
-코드 길이 감소 (클래스의 재사용)
-유지 보수성 증가

상속: 부모의 것을 자식이 물려받는 것(부모의 코드를 자식이 물려받는 것)
: 부모 클래스가 가지고 있는 멤버들을 새로 작성할 클래스를 직접 만들지 않고,
상속을 받음으로 자식 클래스가 자신의 멤버처럼 사용할 수 있음.
	 => 자식의 몸이 커진다 == 확장한다
	 => 여러 명에게 상속 가능함

[사용법]
[접근제한자]class 클래스명 extends 클래스명{}
Ex) public class Academy(자식) extends Company(부모){...}

-자바에서는 단일 상속만 지원한다. 

컬렉션이란 자바에서 제공하는 자료구조를 담당하는 프레임 워크이다.
List : 자료들을 순차적으로 나열한 자료구조, 인데스로 관리, 중복 객체 가능
Set(hash함수를 사용하는 것이 가장 빠름), 저장 순서 유지x, 중복 객체 저장x 
Map: 키와 값으로 구성(모두 객체),중복 저장x, 중복의 경우 키가 덮어쓰기함
